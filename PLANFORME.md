# 🧭 5-Month HLD Roadmap (Backend/System Design Focus)

### 🎯 Purpose
This roadmap is designed for a **Java backend developer (3+ years experience)** to master High-Level Design (HLD), build scalable systems, and crack system design interviews.

🕒 Recommended time: **1.5–2 hours/day (parallel with DSA + LLD)**  
📌 Approach: **1 system design problem per week + deep dive + iteration**

---

## 📅 Month 1 — HLD Foundations (Scalability Basics)

**🎯 Goal:** Understand how large-scale systems are structured.

**📘 Topics**
- What is HLD vs LLD
- Scalability (Vertical vs Horizontal)
- Load Balancing
- Caching Basics (Redis)
- CAP Theorem
- Latency vs Throughput

**🧩 Weekly Problems**
- Week 1: Design URL Shortener (TinyURL)
- Week 2: Design Pastebin
- Week 3: Design Rate Limiter (HLD view)
- Week 4: Design API Gateway

**🏁 Milestone**
> Understand core building blocks of scalable systems.

---

## 📅 Month 2 — Databases & Storage Systems

**🎯 Goal:** Master data storage and retrieval strategies.

**📘 Topics**
- SQL vs NoSQL
- Database Indexing
- Sharding & Partitioning
- Replication (Master-Slave)
- Consistency Models

**🧩 Weekly Problems**
- Week 5: Design User Service (Auth System)
- Week 6: Design Product Catalog (E-commerce)
- Week 7: Design Log Storage System
- Week 8: Design Distributed Cache

**🏁 Milestone**
> Choose the right database and scale data effectively.

---

## 📅 Month 3 — Distributed Systems & Communication

**🎯 Goal:** Build resilient and distributed architectures.

**📘 Topics**
- Microservices Architecture
- REST vs gRPC
- Message Queues (Kafka, RabbitMQ)
- Event-driven systems
- Circuit Breaker Pattern

**🧩 Weekly Problems**
- Week 9: Design Notification System
- Week 10: Design Chat System (WhatsApp)
- Week 11: Design Order Processing System
- Week 12: Design File Upload System

**🏁 Milestone**
> Design systems that scale across multiple services.

---

## 📅 Month 4 — Performance & Reliability

**🎯 Goal:** Optimize systems for scale and fault tolerance.

**📘 Topics**
- CDN (Content Delivery Network)
- Caching Strategies (Write-through, Write-back)
- Rate Limiting at scale
- Fault Tolerance & Failover
- Monitoring & Logging

**🧩 Weekly Problems**
- Week 13: Design Video Streaming System (YouTube)
- Week 14: Design News Feed (Facebook/Instagram)
- Week 15: Design Ride Sharing System (Uber)
- Week 16: Design Search Autocomplete

**🏁 Milestone**
> Handle high traffic and failures gracefully.

---

## 📅 Month 5 — Advanced System Design & Interviews

**🎯 Goal:** Think like a senior/backend architect.

**📘 Topics**
- Distributed Transactions
- Eventual Consistency
- Idempotency
- API Design Best Practices
- Trade-offs in System Design

**🧩 Weekly Problems**
- Week 17: Design Food Delivery System (Swiggy/Zomato)
- Week 18: Design Payment System
- Week 19: Design Online Multiplayer Game
- Week 20: Design Global Distributed System (Google Docs)

**🏁 Milestone**
> Confidently handle real-world system design interviews.

---

## 🧱 Weekly Routine

| Day | Focus |
|-----|------|
| Mon | Understand problem + clarify requirements |
| Tue | Define high-level components |
| Wed | Design data flow & APIs |
| Thu | Deep dive into database & scaling |
| Fri | Identify bottlenecks + optimizations |
| Sat | Mock interview (explain system) |
| Sun | Revise + improve design |

---

## 🧩 Key Concepts to Practice Every Week

- Requirement clarification (functional + non-functional)
- Back-of-the-envelope estimation
- API design
- Database schema design
- Scaling strategies
- Trade-off discussions

---

## 🧩 Supplementary Skills

- Learn basics of **Redis, Kafka, Docker**
- Read system design case studies
- Explore real-world architectures (Netflix, Amazon)
- Practice diagramming (draw.io)

---

## 🚀 Final Outcome After 5 Months

✅ Design **large-scale distributed systems confidently**  
✅ Handle **system design interview rounds**  
✅ Understand **scalability, reliability, trade-offs**  
✅ Think like a **senior backend/system architect**
